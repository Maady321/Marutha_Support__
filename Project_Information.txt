2. HOW THIS PROJECT WORKS
-------------------------
The project uses a modern "Client-Server" architecture but is written in a simple way for students to understand.

*   BACKEND (The Brain): 
    - Built using FastAPI (Python).
    - It handles database connections, security (creating tokens for login), and real-time messaging.
    - Files in the 'backend' folder control the logic (Auth, Users, Clinical records).

*   FRONTEND (The Face): 
    - Built using Vanilla HTML, CSS, and JavaScript.
    - No complex frameworks (like React) are used, making it easier to study.
    - JavaScript files in 'frontend/static/js' talk to the Backend via "Fetch" calls to get and send data.

*   DATABASE: 
    - Uses PostgreSQL (can also use SQLite).
    - Stores information about Users, Patient Records, Vitals, and Chat Messages.

3. FEATURES OF THE WEBSITE
--------------------------
The website is a support platform for palliative care, divided into three main roles:

A. PATIENT DASHBOARD
   - Dashboard Overview: View current health status and recent updates.
   - Vitals Tracking: Log and monitor health stats like blood pressure, heart rate, etc.
   - Communication: Chat directly with assigned Doctors and Volunteers.
   - Medical History: Access past records and prescriptions.

B. DOCTOR DASHBOARD
   - Patient Management: View all assigned patients and their health history.
   - Clinical Records: Create and manage health records, notes, and prescriptions.
   - Appointment Requests: Approve or manage patient consultations.
   - Volunteer Coordination: Assign volunteers to specific patients.

C. VOLUNTEER DASHBOARD
   - Supportive Care: Manage a list of assigned patients to provide non-medical support.
   - Activity Logging: Track daily or weekly volunteer activities performed for patients.
   - Quick Communication: Talk to both the patient and the doctor.

D. GENERAL FEATURES
   - Advanced Auth: Secure Registration and Login with role-based access.
   - Responsive Design: Works on any screen size with a mobile-friendly navigation "hamburger" menu.
   - Real-time Chat: Instant messaging between users.
   - Report Management: Upload and view medical reports.
